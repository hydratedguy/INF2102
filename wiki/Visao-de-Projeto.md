# Visão de Projeto

## Objetivo

O SimBio foi desenvolvido para preencher uma lacuna no mercado brasileiro de biometano: a falta de ferramentas acessíveis para análise rápida de viabilidade de projetos. A ferramenta visa democratizar o acesso a análises financeiras que anteriormente exigiam consultoria especializada.

## Propósito dos Cenários

Seguindo a metodologia de Design Baseado em Cenários de [John Carroll (2000)](https://mitpress.mit.edu/9780262513883/making-use/), utilizamos cenários como histórias concretas de uso que guiam o design, a utilização e a evolução do sistema. Os cenários ajudam a:

- **Compreender contextos de uso**: Entender como diferentes usuários interagem com o sistema em situações reais
- **Identificar requisitos**: Descobrir funcionalidades necessárias através de narrativas de uso
- **Antecipar problemas**: Cenários negativos revelam pontos de falha e oportunidades de melhoria
- **Comunicar visão**: Facilitar o alinhamento entre stakeholders sobre o que o sistema deve fazer

## Cenários de Uso

### Cenários Positivos
Descrevem situações onde o sistema atende com sucesso às necessidades dos usuários:

1. **[Cenário Positivo 1 - Análise de Viabilidade por Investidor](Cenario-Positivo-1)**
   Um investidor utiliza o SimBio para avaliar rapidamente a viabilidade de um projeto de biometano, tomando uma decisão informada sobre alocação de capital.

2. **[Cenário Positivo 2 - Comparação de Rotas Logísticas](Cenario-Positivo-2)**
   Um consultor de energia compara diferentes configurações de transporte para otimizar custos logísticos de um projeto existente.

### Cenários Negativos
Descrevem situações problemáticas que revelam limitações ou áreas de melhoria:

1. **[Cenário Negativo 1 - Dados Incompletos no Mapa](Cenario-Negativo-1)**
   Um usuário não encontra a fonte de matéria-prima desejada no mapa, revelando a necessidade de atualização constante da base de dados.

2. **[Cenário Negativo 2 - Resultado Financeiro Inviável](Cenario-Negativo-2)**
   Um usuário obtém resultado de projeto inviável mas não consegue identificar quais parâmetros ajustar para melhorar a viabilidade.
